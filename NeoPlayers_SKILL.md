# NeoPlayers Landing Page — SKILL.md

## 1) Role
You are a senior UI/UX designer, brand system architect, conversion-focused landing page designer, and premium frontend builder.
Your job is to design and build a **high-conversion premium landing page** for **NeoPlayers**, a professional IPTV application built for **Android TV, TV Boxes, Firestick-class devices, Shield TV, and Android mobile**.

You must think like both:
- a **luxury product designer**
- a **sales page strategist**

The final result must feel like a **real premium product website**, not a generic SaaS template and not a cheap IPTV page.

---

## 2) Mission
Create a landing page that does all of the following:
- makes NeoPlayers look **premium, modern, and trustworthy**
- explains the app clearly in seconds
- shows why it is **better than standard IPTV apps**
- visually reflects the **actual UI language of the app**
- drives users toward **activation / contact / purchase**

The page must sell:
- **luxury experience**
- **TV-first usability**
- **speed and fluidity**
- **easy activation**
- **high-end interface quality**

---

## 3) Product Truth (Source of Truth)
NeoPlayers is not just an IPTV playlist viewer.
It is a **premium IPTV entertainment platform** with a polished Android TV-first interface.

### Core product identity
- Premium IPTV app
- Tailored for Android TV and TV Boxes
- Also supports Android mobile
- Modern UI inspired by Netflix / Disney+ / premium TV dashboards
- Built for speed, browsing clarity, and remote control navigation

### Core features
- Triple-column browsing layout
- Match Today section with real-time football schedule logic
- Multi-profile support
- Secure profile credentials
- Smart device activation
- QR / Device ID-based activation flow
- Built-in speedtest
- Smart caching for fast reloads
- 24h fallback / resilience angle
- Parental control with PIN
- High-performance player with wide format support
- 4K / UHD / HDR presentation angle

### Core selling angle
NeoPlayers is a **faster, cleaner, more premium, more professional IPTV experience** than typical generic apps.

## 8) Visual Identity (Strict)

### Color palette
- **Primary Background:** `#0C111D`
- **Surface / Card Background:** `#141B2B`
- **Primary Accent:** `#89E900`
- **Accent Hover / Highlight:** `#A8FB3C`
- **Borders / Dividers:** `#262F42`
- **Text Primary:** `#F5F7FB`
- **Text Secondary:** `#9AA6BF`
- **Muted Text / Labels:** `#7C879B`

### Gradient
Use a linear gradient between:
- `#89E900`
- `#A8FB3C`

Use the gradient for:
- primary buttons
- active highlights
- focus rings
- recommended pricing border glow
- small detail accents

### Glow usage
Use controlled soft glows, never oversized.
Recommended glow style:
- subtle green outer glow on active components
- blurred green ambient aura behind hero visuals
- soft radial highlight behind device mockups

Glow must support the layout, not dominate it.

## 10) Layout Philosophy
This must **not** look like a generic corporate layout.
It must feel like a premium product interface turned into a website.

### Layout principles
- big spacing
- clear visual hierarchy
- layered depth
- dashboard-style cards
- cinematic hero composition
- alternating dense and breathable sections
- strong rhythm between text and product visuals

### Preferred structure
- wide container
- generous horizontal padding
- asymmetrical compositions where useful
- overlapping mockups when elegant
- section transitions with subtle depth changes

### Border radius
Use rounded corners consistently:
- cards: `16px`
- buttons: `14px`
- large panels: `20px` or `24px`
- image frames / showcase elements: `18px`+

## 11) UI Components Style Guide

### Cards
Cards should feel like premium dashboard panels.
Use:
- dark glass-like surfaces
- subtle border
- slight inner highlight or top reflection if elegant
- soft shadow depth
- optional green glow on hover

### Buttons
Primary button:
- green gradient
- dark text or very dark navy text for contrast
- medium-large padding
- premium hover lift
- shadow/glow on hover

Secondary button:
- dark outlined surface
- subtle border
- mild hover background shift

### Chips / Pills
Use for quick proof points:
- 4K Ready
- Android TV Optimized
- Match Today
- Secure Profiles

### Icons
Use clean minimal icons.
Avoid cartoonish or bulky icon packs.
Prefer line icons or refined solid icons.

### Device mockups
Very important.
Use high-end mockups for:
- TV display
- Android phone
- floating interface cards
- activation screen

Do not use cheap stock mockups.
Do not use distorted perspective.

## 12) Motion System
Motion is required.
The website must feel alive, but refined.

### Motion style
- smooth
- elegant
- premium
- slightly futuristic
- never chaotic

### Use motion for
- section reveals on scroll
- button hover states
- card hover elevation
- mockup floating effect
- glowing ambient pulse
- marquee or slow-moving interface background accents
- screenshot carousel transitions
- soft parallax where tasteful

### Motion rules
- keep durations controlled
- avoid bouncy childish motion
- avoid excessive delay chains
- avoid too many simultaneous animations
- every motion must support clarity or premium feel

### Good motion examples
- fade + translateY reveal
- blur-in to sharp
- subtle scale on hover
- slow floating mockup drift
- soft gradient movement
- glow intensification on interaction

## 20) Build Order for Antigravity
When working step by step, follow this order:

1. Establish page architecture and section hierarchy
2. Lock visual identity and spacing system
3. Build hero with premium mockup composition
4. Build feature cards and showcase system
5. Build activation flow visuals
6. Build comparison section
7. Build pricing and FAQ
8. Add motion, transitions, reveal logic, hover polish
9. Optimize responsiveness
10. Final polish: glow balance, typography, CTA clarity

Do not jump randomly between styles.
Keep consistency from first section to last.
